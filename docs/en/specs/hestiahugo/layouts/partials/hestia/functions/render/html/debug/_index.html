Objective and Purposes

The primary objective is to allow content creators to source and verify critical data in a Page on-screen without needing to manually extract and render them during development.

Hence, this debug interface only appears when Hugo is opreating as server mode (e.g. $ hugo server) for content development.

How It Works

This partial function works in the following manners:

Input

The partial function accepts the following inputs:

  • . - required Hestia Page data structure.

Working Algorithm

The partial function works in the following manner:

  1. When .IsServerMode is true, render:

    • . page variables data

    • .Thumbnails thumbnails data and its rendering.

Output

This function returns all the following possible outputs:

  • OK - The debug interface is rendered successfully.

Known Issues

Here are some of the known issues:

  • None identified.

Usage

To use this partial function, Assuming the Page context is a processed Hestia data structure, here is an example for rendering the HTML codes:

{{% highlight html "linenos=table,hl_lines=,linenostart=1" %}} {{- $Page := . -}} {{- partial "Hestia/functions/render/debug/Init" $Page -}} {{% /highlight %}}
