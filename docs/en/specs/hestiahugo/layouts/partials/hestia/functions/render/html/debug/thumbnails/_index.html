Objective and Purposes

The primary objective is to make .Thumbnails variables data easily accessible to content creators for visual debugging.

This debug interface only appears when Hugo is opreating as server mode (e.g. $ hugo server) for content development.

How It Works

By default, this function renders the entire .Thumbnails variables into a table of key:value list, allowing one to check all the rendered thumbnails data.

Input

This partial function accepts the following inputs:

  • . - required Hestia Page data structure.

Working Algorithm

This partial function works in the following manner:

  1. When .IsServerMode value is true, render the entire .Thumbnails page into a list of KEY:Value table.

Output

This function returns all the following possible outputs:

  • OK - The page variable is rendered.

Known Issues

Here are some of the known issues.

  • None identified.

Usage

To use this partial function, Assuming the Page context is a processed Hestia data structure, here is an example for rendering the HTML meta title:

{{% highlight html "linenos=table,hl_lines=,linenostart=1" %}} {{- $Page := . -}} {{- partial "Hestia/functions/render/html/debug/Thumbnails" $Page -}} ... {{% /highlight %}}
