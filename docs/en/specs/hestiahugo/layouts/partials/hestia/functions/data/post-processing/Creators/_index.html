Objective and Purposes

The primary objective is to process all Hestia's Page Creators data into its final form, ready for use.

How It Works

By default, the algorithm loop through the .Page.Creators lists, and send each Creator data to }}">Post-Processing/Creator partial function for post-processing. Then, the algorithm forms a new list and updates .Page.Creators with the finalized data structure.

Input

This partial function accepts the following inputs:

  • .Page - required Hestia Page data structure.

  • .Page.Creators - the raw Hestia Creators data structure list for post-processing.

Working Algorithm

This partial function works in the following manner:

  1. Loop through each Creators list, then each of its Creator data in .Page.Creators and send it to }}">Post-Processing/Creator partial function for post-processing.

  2. If the Creator data is available (and not discarded), append the processed data into the new list.

  3. Update .Input.Creators with the new list.

Output

This function returns all the following possible outputs:

  • OK - The Creators data list has been successfully processed.

  • ERROR - bad media type - one or more given Creator data has missing critical fields.
