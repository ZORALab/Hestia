Objective and Purposes

The primary objective is to process all Hestia's Page Keywords data into its final form, ready for use.

How It Works

By default, the algorithm loop through each keyword from .Keywords and send it to }}">Hestia GoTemplateFunction Data Scanner partial function for processing. Then, it is sent to to }}">Hestia Whitespace Trimming partial function for clean up before merging into a key:true-value list for uniqueness sorting.

The algorithm ends by processing the key:true-value list back to array slice and update the page data.

Input

This partial function accepts the following inputs:

  • . - required Hestia Page data structure.

Working Algorithm

This partial function works in the following manner:

  1. Get a .Keyword from the .Keywords slice list and process it with }}"> Hestia GoTemplateFunction Data Scanner partial function.

  2. Clean the processed keyword using }}"> Hestia Trim Whitespace partial function.

  3. Save the processed and cleaned keyword into a key:true-value dictionary list for uniqueness sorting. Repeat step 1 until the entire .Keywords list is completed.

  4. Convert the key:true-value dictionary list back into array slice list before returning as output.

Output

This function returns all the following possible outputs:

  • OK - The titles data list has been successfully processed.

  • ERROR - bad formatting - one or more keyword data has a malformed GoTemplateFunction formatting clause.
