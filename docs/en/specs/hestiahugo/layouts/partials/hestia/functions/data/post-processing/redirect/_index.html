Objective and Purposes

The primary objective is to process all Hestia's Page Redirect data into its final form, ready for use.

How It Works

By default, the algorithm runs through each element of the .Redirect data. It first processes the .Redirect.URL with }}">Hestia URL Partial Shortcode. It then processes the .Redirect.Delay data and .Redirect.Enabled for type checking and their value's checking. Once done, the algorithm determines the enabling is valid and shall append the necessary page robots data accordingly. Otherwise, by default, regardless of any error occured, the redirect shall always be disabled.

Input

This partial function accepts the following inputs:

  • . - required Hestia data structure.

Working Algorithm

This partial function works in the following manner:

  1. Process the .Redirect.URL with }}">Hestia URL Partial Shortcode.

  2. Process the .Redirect.Delay against its int* type and its value validity.

  3. Process the .Redirect.Enabled data.

  4. Process the necessary configurations (e.g. adding robots:noindex data field) when .Redirect.Enabled is true.

Output

This function returns all the following possible outputs:

  • OK - The redirect data structure has been successfully processed.

  • ERROR - invalid delay data type - The given .Redirect.Delay is not a number. Remove quotation if you're providing it as a string.

  • ERROR - bad/invalid url - The given .Redirect.URL is has bad or invalid.

  • ERROR - enabled but missing URL - The .Redirect.Enabled is set to enabled but the given .Redirect.URL is missing.
