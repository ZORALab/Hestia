Objective and Purposes

The primary objective is to systematically and uniformly renders the page title according to a given pattern.

This is primarily used for HTML codes in <head> section.

How It Works

By default, the function obtains a standard format from data/Hestia/Internal/Meta.toml - Title.HTML field. It then checks the .Titles.Pattern overrides availability and will use the latter if available. Should both of them are unavailable, the function resorts to the default "{{- .Titles.Page -}}" format.

Once done, it shall process with }}">HestiaHUGO's Go Template Function Data Scanner Partial Functions to process the string. It shall render the <title> tag as a result without any return values.

Input

This partial function accepts the following inputs:

  • .Page - required Hestia Page data structure.

Working Algorithm

This partial function works in the following manner:

  1. Obtain the default format from .Data.Hestia.Internal.Meta.Title.HTML field. Check for .Titles.Pattern format overrides and if available, take the latter instead. If both are unavailable, use the built-in default format ("{{- .Titles.Page -}}").

  2. Process the obtained format string with }}">HestiaHUGO's Go Template Function Data Scanner Partial Functions.

  3. Render the title in HTML format.

Output

This function returns all the following possible outputs:

  • OK - HTML title code rendered with values.

  • ERROR - missing partial function - a template clause calls a missing or invalid Hestia partial shortcode function. Returned string is unusable and chaotic in nature.

  • ERROR - improper clauses - a template clause is usable but the clause pattern does not conforms with Go Template formatting pattern. Returned string is usable if good since this error is a notice and it is not a rejection.
