Objective and Purposes

The primary objective is to process all Hestia's Page Titles data into its final form, ready for use.

How It Works

By default, the algorithm first processes .Titles.Site with }}">Hestia GoTemplateFunction Data Scanner partial function. Then, the same process is repeated for .Titles.Page.

.Titles.Site goes first because it can be used for value insertion for .Titles.Page for some users (e.g. appending a tailing | {{ .Titles.Site -}} pattern.

Input

This partial function accepts the following inputs:

  • . - required Hestia Page data structure.

Working Algorithm

This partial function works in the following manner:

  1. Process .Titles.Site with }}"> Hestia GoTemplateFunction Data Scanner partial function.

  2. Process .Titles.Page with }}"> Hestia GoTemplateFunction Data Scanner partial function.

Output

This function returns all the following possible outputs:

  • OK - The titles data list has been successfully processed.

  • ERROR - bad formatting - one or more given title data has a malformed GoTemplateFunction formatting clause.
