Objective and Purposes

The primary objective is to process all Hestia's Page Descriptions data into its final form, ready for use.

How It Works

By default, the algorithm runs through each description data in the .Page.Descriptions data list. It first select .Page.Descriptions.Pitch and send it to }}">Hestia GoTemplateFunction Data Scanner partial function. It then send it to }}">Hestia Trim Data Whitespace partial function for clean up. It then replaces all new lines into space to ensure the entire description is a single paragraph. Lastly, it checks the description against its max character limits.

Input

This partial function accepts the following inputs:

  • . - required Hestia Page data structure.

Working Algorithm

This partial function works in the following manner:

  1. Send .Descriptions.Pitch into }}">Hestia GoTemplateFunction Data Scanner partial function.

  2. Send the processed data into }}">Hestia Trim Data Whitespace partial function for clean up.

  3. Replace all new lines in the between the processed data into space to make sure the entire data is a single paragraph.

  4. Count its rune characters against its 160 max characters limit. If exceeded, a console error is then raised.

  5. Repeat step 1-4 for .Descriptions.Summary against its 250 max characters limit.

Output

This function returns all the following possible outputs:

  • OK - The descriptions data list has been successfully processed.

  • ERROR - bad formatting - one or more given description data has a malformed GoTemplateFunction formatting clause.

  • ERROR - max limit - one or more given description data exceeded its maximum rune characters limit.
