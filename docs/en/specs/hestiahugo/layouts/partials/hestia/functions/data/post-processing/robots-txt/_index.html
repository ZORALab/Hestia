Objective and Purposes

The primary objective is to process all Hestia's Page RobotsTXT data into its final form, ready for use.

How It Works

By default, the algorithm runs through each Robots data in the RobotsTXT data list. It processes the .Sitemaps with unique list, absolute URL processing against the given Base URL, and whitespace cleaning with }}">Hestia Trim Data Whitespace partial function. It repeats the process for all relative URLs in .Allow and .Disallow linked list. Lastly, it processes Crawl-delay field with the same algorithm.

Input

This partial function accepts the following inputs:

  • . - required Hestia's RobotsTXT data structure.

Working Algorithm

This partial function works in the following manner:

  1. Process .Name of a user agent by replace All and all into the correct * user-agent value. Then, it sends the processed data into }}">Hestia Trim Data Whitespace partial function for clean up before saving it as the final output.

  2. Loop through .Allow and for each value, it is processed with {{ .BaseURL }} replacement, clean up with }}">Hestia Trim Data Whitespace partial function and save it as in a new list. The list is then updated to replace outdated .Allow list.

  3. Loop through .Disallow and for each value, it is processed with {{ .BaseURL }} replacement, clean up with }}">Hestia Trim Data Whitespace partial function and save it as in a new list. The list is then updated to replace outdated .Disallow list.

  4. Get the .Crawl-delay value and send it for clean up with }}">Hestia Trim Data Whitespace partial function. The processed data is then updated.

  5. Repeat Step 1 for other user agent dataset.

Output

This function returns all the following possible outputs:

  • OK - The RobotsTXT data structure has been successfully processed.
