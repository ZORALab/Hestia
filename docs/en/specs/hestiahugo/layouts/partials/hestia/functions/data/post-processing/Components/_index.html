Objective and Purposes

The primary objective is to process all Hestia's Page Component data into its final form, ready for use.

How It Works

By default, the algorithm loop through the .Page.Components data list and send each Component data to }}">Post-Processing/Component partial function for post-processing. Then, the algorithm forms a new list and updates .Page.Components with the finalized data structure.

Input

This partial function accepts the following inputs:

  • .Page - required Hestia Page data structure.

  • .Page.Components - the raw Hestia Components data structure list for post-processing.

Working Algorithm

This partial function works in the following manner:

  1. Loop through each thumbnail data in .Page.Components and send it to }}">Post-Processing/Component partial function for post-processing.

  2. If the Component data is available (and not discarded), append the processed data into the new list.

  3. Update .Input.Components with the new list.

Output

This function returns all the following possible outputs:

  • OK - The components data list has been successfully processed.

  • ERROR - unknown HestiaUI component - one or more of the included component is not available.
