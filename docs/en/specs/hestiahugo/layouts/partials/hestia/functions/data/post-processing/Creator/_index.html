Objective and Purposes

The primary objective is to process a given Hestia's Page Creator data into its final form, ready for use.

How It Works

By default, the algorithm process each fields of a given Creator data. It shall process all the URLs with }}">Hestia URL partial shortcode. It also process .Description with }}">Hestia GoTemplateFunction Data Scanner partial function.

The algorithm SHALL discard the entire creator data list when .Type and .Name empty.

Input

This partial function accepts the following inputs:

  • .Page - required Hestia Page data structure.

  • .Input.Type - the creator type like person and organization.

  • .Input.Name - the creator's (displaying) full name.

  • .Input.Title - the creator's associated branding like job title or company brand tag.

  • .Input.Phone - the creator's contactable phone number.

  • .Input.Email - the creator's contactable email address.

  • .Input.LinkedIn - the creator's LinkedIn URL.

  • .Input.Facebook - the creator's Facebook URL.

  • .Input.Instagram - the creator's Instagram URL.

  • .Input.Twitter - the creator's Twitter URL.

  • .Input.TikTok - the creator's TikTok URL.

  • .Input.Web - the creator's Website URL.

  • .Input.Description - the creator's short description.

Working Algorithm

This partial function works in the following manner:

  1. Process the .Input.Type. It shall identify and covert its value to lowercase.

  2. Copy over .Input.Name.

  3. Copy over .Input.Phone.

  4. Copy over .Input.Email.

  5. Process the .Input.LinkedIn with }}"> Hestia URL partial shortcode.

  6. Process the .Input.Facebook with }}"> Hestia URL partial shortcode.

  7. Process the .Input.Instagram with }}"> Hestia URL partial shortcode.

  8. Process the .Input.Twitter with }}"> Hestia URL partial shortcode.

  9. Process the .Input.TikTok with }}"> Hestia URL partial shortcode.

  10. Process the .Input.Web with }}"> Hestia URL partial shortcode.

  11. Process the .Input.Description with }}"> Hestia GoTemplateFunction Data Scanner partial function.

  12. Remove the Creator data if any of the following are empty:

    • .Type

    • .Name

  13. Return the processed data.

Output

This function returns all the following possible outputs:

  • OK - The contact data has been successfully processed.

  • ERROR - missing critical fields - Given creator data is having 1 or more missing critical fields. An empty data structure is returned.
