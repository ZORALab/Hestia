Multiple Licenses

ZORALab's Hestia Project is a wide coverage software library mono repository consisting of codes, non-code media files, code-based media files, configuration files, automation config files, and 3rd-party software for automation purposes. Therefore, the project must issue multiple licenses for various types of products and assets for difference use cases.


Code-based Products

Generally speaking, ZORALab's Hestia Project produces a suite of code-based software libraries usually written in their formatted files including code-based artworks like SVG vector graphic files. These coded products are licensed under the OSI Approved Apache 2.0 License.



Please DO:

  • Credit back to ZORALab's Hestia Project when used in yours like how we credits all the copyright holders and contributors.

  • Use in your commerical project as long as copyright accreditation is honored.

  • If contributing back: ensures your contributions patches are readily compatible with Apache 2.0 license.

  • When contributing back: present the referenced sources of your contributions if any (we do add them into the files' copyright header).

  • Respect the licensed and copyright terms and conditions of all your references.



Please DO NOT:

  • Use ZORALab's Hestia in your project and then claim the entire project is build under your novel development. (Reason: breach of license's terms and disrespect all the copyright holders and contributors in the Project).

  • Copy pasting your codes from somewhere without referencing or tracking your sources.

  • Reference any open-source source codes that has copyleft effect like GPL-2 licensed codes (incompatible with Apache 2.0).

  • Reference any proprietary or close-source codes that are not readily available for open-sourcing and re-licensing to Apache-2.0.

  • Disrespect your reference's copyright, license's terms, or even their privacy.


Media-based Materials

While documentations may use non-code based media materials like rasterized photo images, video, or audio of any kinds, to prevent ZORALab's Hestia Project from being a re-distributor, you are prohibited from using them directly from us. However, we always credit the original artists so you can source their artworks directly from them.



Please DO:

  • Trace the artwork back to the original artists and source it from the principle owner.

  • Ask us if you cannot find the original artist.



Please DO NOT:

  • Use any of the non-code based assets (images, videos, audio, etc) in any of your project.

  • Use any of the non-code based assets and credit back to ZORALab's Hestia Project.
