Business Direction

As the name implies, this section is all about assimilating or developing new or existing human interactive functional user-interfaces deployed across various technologies. The idea is to secure and to facilitate all available functional UI artwork to an existing and future projects use.

In short, this is the programmable artwork division of the ZORALab's Hestia Project.

Compulsory Requirements

After the first successful HestiaUI implementations, there are some known requirements that are non-compromising in order to keep ZORALab's Hestia project maintainable.

Always Componentize

There are too many user interfaces in this world contributing to different forms of interactions. Some are context-dependent while some are uniquely tied to an art direction. To manage them in the same repository can be a daunting task. In any case, every UI parts MUST ALWAYS be a component stating its inter-dependencies of one another.


Secondly, the only sensible way of testing UI is having human to test it. Therefore, keeping the UI as a small and modular compoment makes regression testing a lot easier to manage with.

Regressively Testable

ZORALab's Hestia is a continuously improved project: testing regressively is a staple practice here. This includes art-related UI design work.


Currently, ZORALab's Hestia had chosen Hugo as its primary engine to develop and test graphical UI designs. Hence, please ensure all UI designs are visually and interactively tested by humans.

Avoid Copyrightable Multimedia Dependencies

Unless you plan to sponsor the IP attorney fees for processing the re-distributable multimedia contents (e.g. picture, video, audio, etc.) written licenses (e.g. individual model release form, unlimited publications and redistribution right agreement, etc), please forbid any component design from being locked to 1 or more media file(s).


You can either contribute media files that are code based (e.g. svg files) compatible with ZORALab's licenses OR document the instructions and let user to fill in his/her own media materials. DO NOT REDISTRIBUTE UNAUTHORIZED OR WORSE, UNLICENSED COPYRIGHTED MATERIALS.

Maximizing Technological Portabilities

UI is an art; it has to be portable in nature. Folks will use this project for various kinds of deployment and use cases. At least for web technologies: CSS-only, CSS+JS, and CSS+WASM use cases are supported by ZORALab's Hestia.


So for the next component design, please keep it portable and translatable from one technology to another easily.

Keep Innovation Door Open

As the next generation artificial intelligence technologies are on the rise for generative applications, ZORALab's Hestia intend to embrace and welcome new AI technologies.


Please keep the door open for new techniques and ideas from the new technologies.

Avoid Vendor Locked-In

Please avoid vendor-specific features (e.g. Hugo shortocode) whenever possible. Your UI design shall be implementable across various technologies not limited to web only (e.g. Qt5).
